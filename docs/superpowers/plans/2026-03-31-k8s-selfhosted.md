# K8s Self-Hosted Services Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy n8n, Grafana, Vault, and Qdrant on Rackspace Kubernetes Engine using Helm+Kustomize manifests delivered via FluxCD GitOps.

**Architecture:** Per-service namespaces with FluxCD `Kustomization` CRs enforcing bootstrap order via `dependsOn`. Helm charts rendered by Flux. Vault (Raft mode) is the secret store; External Secrets Operator syncs secrets into app namespaces. NGINX Ingress + cert-manager provides TLS via Let's Encrypt for all four services at `*.quybits.com`.

**Tech Stack:** Kubernetes (Rackspace RKE), FluxCD v2, Helm v3, Kustomize, Vault (Raft), External Secrets Operator v0.9+, NGINX Ingress Controller, cert-manager v1.14+, StorageClass `ssd`, kubeconform (validation)

---

## Prerequisites (manual, before Task 1)

- `kubectl` connected to Rackspace cluster: `kubectl cluster-info`
- `flux` CLI installed: `brew install fluxcd/tap/flux`
- `helm` CLI installed: `brew install helm`
- `kubeconform` installed: `brew install kubeconform`
- GitHub repo created for this project (used as FluxCD source)
- DNS A records: `n8n`, `grafana`, `vault`, `qdrant` subdomains of `quybits.com` → cluster LoadBalancer IP (can be done after Task 10 once the IP is known)

- [ ] **Step 2: Create bootstrap Job**

Create `infrastructure/vault/bootstrap-job.yaml`:

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: vault-bootstrap
  namespace: vault
spec:
  ttlSecondsAfterFinished: 600
  template:
    spec:
      serviceAccountName: vault-bootstrap
      restartPolicy: OnFailure
      initContainers:
        - name: install-kubectl
          image: alpine:3.19
          command:
            - sh
            - -c
            - |
              apk add --no-cache curl
              curl -LO "https://dl.k8s.io/release/v1.29.0/bin/linux/amd64/kubectl"
              chmod +x kubectl
              cp kubectl /tools/kubectl
          volumeMounts:
            - name: tools
              mountPath: /tools
      containers:
        - name: bootstrap
          image: hashicorp/vault:1.17
          env:
            - name: VAULT_ADDR
              value: "http://vault.vault.svc.cluster.local:8200"
            - name: VAULT_TOKEN
              valueFrom:
                secretKeyRef:
                  name: vault-root-token
                  key: token
          volumeMounts:
            - name: tools
              mountPath: /tools
          command:
            - /bin/sh
            - -c
            - |
              set -e
              export PATH="$PATH:/tools"

              # Wait for Vault to be unsealed
              echo "Waiting for Vault to be unsealed..."
              until vault status 2>/dev/null | grep -q "Sealed.*false"; do
                echo "  vault sealed or unreachable, retrying in 10s..."
                sleep 10
              done
              echo "Vault is unsealed."

              # Enable KV v2 at secret/
              vault secrets enable -path=secret kv-v2 2>/dev/null \
                || echo "KV v2 already enabled at secret/"

              # Enable AppRole auth
              vault auth enable approle 2>/dev/null \
                || echo "AppRole already enabled"

              # Policy: ESO reads all secrets
              vault policy write eso-policy - <<'POLICY'
              path "secret/data/*"     { capabilities = ["read", "list"] }
              path "secret/metadata/*" { capabilities = ["read", "list"] }
              POLICY

              # Policies scoped per app
              vault policy write n8n-policy - <<'POLICY'
              path "secret/data/n8n" { capabilities = ["read"] }
              POLICY

              vault policy write grafana-policy - <<'POLICY'
              path "secret/data/grafana" { capabilities = ["read"] }
              POLICY

              vault policy write qdrant-policy - <<'POLICY'
              path "secret/data/qdrant" { capabilities = ["read"] }
              POLICY

              # AppRole for ESO
              vault write auth/approle/role/eso \
                token_policies="eso-policy" \
                token_ttl=1h \
                token_max_ttl=24h

              ROLE_ID=$(vault read -field=role_id auth/approle/role/eso/role-id)
              SECRET_ID=$(vault write -f -field=secret_id auth/approle/role/eso/secret-id)

              # Write AppRole credentials to external-secrets namespace
              kubectl create secret generic vault-approle-credentials \
                -n external-secrets \
                --from-literal=roleId="${ROLE_ID}" \
                --from-literal=secretId="${SECRET_ID}" \
                --dry-run=client -o yaml | kubectl apply -f -

              # Seed placeholder secrets — UPDATE THESE before going live
              vault kv put secret/n8n \
                encryption_key="changeme-replace-with-32-char-key"

              vault kv put secret/grafana \
                admin_password="changeme-replace-before-live"

              vault kv put secret/qdrant \
                api_key="changeme-replace-before-live"

              echo "Bootstrap complete. Update secret values in Vault before going live."
      volumes:
        - name: tools
          emptyDir: {}
```
